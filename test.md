2) Интеграция   с   оракулом   Chainlink   для   получения   актуальных   ценовых

данных   с   централизованных   бирж   (CEX)   для   алгоритмов   комиссий,

PoolManager,   SwapRouter,   PositionManager   для   создания   пулов,

управления ликвидностью и выполнения обменных операций (свопов).

4) Предоставление унифицированного интерфейса для всех реализованных

типов   хуков   динамических   комиссий,

  обеспечивающего   их

совместимость и простоту интеграции.

(Anvil)   с   развертыванием   необходимой   инфраструктуры   и   симуляцией

торговой активности.

6) Развертывание   смарт-контрактов   на   тестовую   сеть   блокчейна   через

специализированные скрипты, включающие подготовку всех зависимых

артефактов.

7) Предоставление   веб-интерфейса   для   конструирования   смарт-контракта

хука на основе выбранного пользователем алгоритма расчета комиссии.

3.2. Описание функционирования программы

Проект реализован как библиотека хуков (смарт-контрактов) для Uniswap V4. Код

смарт-контрактов   и   скрипты   написаны   на   Solidity.   Для   веб-интерфейса   используется

Python, HTML, CSS, JavaScript.

     3.2.1. Смарт-контракты

Основные хуки размещены в папке src/. Пример — src/BAHook.sol. У хуков есть

следующие функции:

- poolManager() — возвращает адрес PoolManager, с которым работает хук

-_getPriceRatio()   —   получает   цены   token0/USDT   и   token1/USDT   из   Chainlink   и

вычисляет отношение token0/token1

-   _afterInitialize()   —   инициализирует   состояние   пула   и   выставляет   начальные

комиссии feeAB и feeBA

- getFee() — возвращает текущую комиссию в зависимости от направления свопа

-   setFee()   —   устанавливает   одинаковую   комиссию   для   обоих   направлений   (этот

метод доступен только для владельца данного пула)

- _afterSwap() — после свопа проверяет, наступил ли новый блок; если да, сравнивает

текущий ценовой коэффициент с предыдущим и изменяет комиссии

     3.2.2. Скрипты развертывания

Скрипт   развертывания   находятся   в   папке   /scripts.   Назначение:   развертывание   пула   и

подключение PriceFeeds для получения актуальных цен на токены. При развертывании на

локальном   блокчейне   скрипт   дополнительно   развертывает   необходимые   артефакты

Uniswap V4 (PoolManager, PositionManager, Router). Логика работы:

– Скрипт наследует контракт HookHelpers, который имеет функции:

1. deployFeeds() — создает mock-фиды для локального блокчейна (вызывается в

конструкторе)

2. deployHook() — по имени хука развертывает нужный контракт.

– Контракт HookHelpers наследуется от контракта BaseScript, в своем конструкторе

создает mock-токены.

– index() — загружает главную HTML-страницу.

– select_hook() — возвращает описание, параметры и код выбранного хука.

Simulation.t.sol.

– generate_swap_graphs()   —   строит   графики   цен   и   соотношений   по   результатам

симуляции.

– parse_sim_output() — парсит вывод симуляции в summary и список свопов.
