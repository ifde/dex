### Notes:

<!-- Slide number: 2 -->
# О чем проект
1️⃣ Проект представляет из себя библиотеку смарт-контрактов (хуков) с динамическими комиссиями для DEX на базе Uniswap V4

2️⃣ Программа осуществляет:
Динамический расчет комиссий пула ликвидности
Интеграцию с оракулом Chainlink
Взаимодействие с артефактами Uniswap V4
Тестирование и симуляцию торговой активности
🌐 Дополнительно: Веб-интерфейс для конструирования хуков

‹#›

### Notes:

<!-- Slide number: 3 -->
# 🎯 Цель
Создать библиотеку смарт-контрактов, которая позволит автоматически корректировать размер комиссии пула ликвидности в зависимости от рыночных условий, снижая непостоянные потери (impermanent loss) для поставщиков ликвидности и повышая эффективность DEX.

‹#›

### Notes:

<!-- Slide number: 4 -->
# 📋 Основные задачи
Реализовать функционал:
✅ Базовое наследование от контракта BaseOverrideFee с унифицированным интерфейсом для всех типов динамических комиссий

✅ Реализация нескольких типов хуков с различными математическими моделями расчета комиссии

‹#›

### Notes:

<!-- Slide number: 5 -->
# 📋 Основные задачи
Реализовать функционал:
✅ Интеграция с оракулом Chainlink для получения актуальных ценовых данных с централизованных бирж (CEX)

✅ Совместимость с ключевыми артефактами Uniswap V4: PoolManager, SwapRouter, PositionManager

✅ Инфраструктура развертывания и тестирования на локальном блокчейне (Anvil)

✅ Веб-интерфейс для конструирования и скачивания смарт-контрактов

<!-- Slide number: 25 -->
# ✅ Смарт-контракты хуков

Основные хуки размещены в папке src/:
- MEVChargeHook.sol — комиссии на основе MEV и арбитражного спреда
- PegStabilityHook.sol — стабилизация цены через корректировку комиссий
- BAHook.sol / DAHook.sol / ABHook.sol — различные алгоритмы расчета

Базовый контракт: BaseOverrideFee.sol — унифицированный интерфейс

‹#›

### Notes:

<!-- Slide number: 26 -->
# 👨‍💻 Реализация функций хука

Основные функции контракта:
- poolManager() — адрес PoolManager
- _getPriceRatio() — получение цен из Chainlink
- _afterInitialize() — инициализация пула и начальных комиссий
- getFee() — текущая комиссия по направлению свопа
- setFee() — установка комиссии владельцем пула
- _afterSwap() — корректировка комиссий после свопа

‹#›

### Notes:

<!-- Slide number: 27 -->
# 👨‍💻 Скрипты развертывания

Скрипты в папке /script:
- DeployHook.s.sol — развертывание хука с адресами фидов
- deployHookAndCreatePool.s.sol — деплой хука и создание пула ликвидности

Иерархия контрактов:
DeployHook → HookHelpers → BaseScript → Deployers

### Notes:

<!-- Slide number: 31 -->
# 🙋‍♂️ Алгоритм работы программы

1) Пользователь выбирает тип хука и параметры на веб-странице
2) Скрипт развертывает артефакты Uniswap V4 и хук на Anvil
3) Загружаются исторические данные Binance
4) Запускается симуляция торговой активности
5) Фиксируются метрики (объем, комиссии, IL)
6) Результаты отображаются в виде графиков в UI
